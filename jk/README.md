1. 全局主导航栏（Global Navbar / Header）
定位：网站首页的顶部导航组件，默认展示在首页最顶端。结构与功能：

视觉特征：白色背景、极简风格、低视觉干扰，适配全页面展示，不抢占内容焦点。
2. 首页 CTA 引导栏（Homepage CTA Bar / Action Banner）
定位：仅在首页首屏展示的转化引导组件，默认隐藏，仅在特定滚动时机触发显示。结构与功能：
左侧：也包含「首页」「岗位洞察」「能力测评」「专属报告」
右侧：强引导主按钮区（「立即测评」按钮，用户点击后跳转进入能力测试(CapabilityAnalysis)界面）
视觉特征：CTA导航栏采用圆角卡片式设计，卡片为白底黑字，其中强引导主按钮区为高对比配色（如蓝底白字）、强视觉冲击力，用于强化用户行动意愿，仅服务于首页转化。

二、切换触发时机
1. 正向切换（主导航收起 → CTA 栏显示）
触发条件：当页面垂直滚动，首页首屏内容完全离开可视区域，滚动位置到达第二个板块顶部时（即滚动距离 ≥ 首屏高度，可通过 window.scrollY ≥ 首屏offsetTop 精准监听），触发切换动画。补充规则：若滚动速度极快，动画仍完整执行，不跳过过渡；若滚动中途停止，动画停留在当前进度，保证视觉连贯。
2. 反向切换（CTA 栏收起 → 主导航显示）
触发条件：当页面向上滚动，滚动位置回到首屏区域，第二个板块完全离开可视区域时（即滚动距离 < 首屏高度），触发反向切换动画，恢复初始状态。
三、切换过程详细动画描述
1. 正向切换动画（主导航 → CTA 栏）
动画总时长：300ms（0.3 秒），采用 ease-in-out 缓动函数，保证过渡自然丝滑。分阶段执行逻辑：
第 0-150ms（同步启动阶段）：
全局主导航栏：以 translateY(-100%) 为目标，向上平滑滑出屏幕，同时透明度从 100% 渐变至 0%，完成「缓缓向上收起」效果；
首页 CTA 引导栏：以 translateY(0) 为目标，从屏幕外（translateY(-100%) 初始位置）向下平滑滑入，同时透明度从 0% 渐变至 100%，完成「慢慢显示」效果。
第 150-300ms（同步收尾阶段）：
主导航栏：完全滑出屏幕，透明度归 0，设置为 visibility: hidden，避免遮挡页面内容；
CTA 引导栏：完全滑入屏幕，透明度归 100%，固定在页面顶部（position: sticky 吸顶），保持可见状态，直至用户反向滚动。
2. 反向切换动画（CTA 栏 → 主导航）
动画总时长：300ms（0.3 秒），缓动函数与正向一致，保证体验统一。分阶段执行逻辑：
第 0-150ms（同步启动阶段）：
首页 CTA 引导栏：以 translateY(-100%) 为目标，向上平滑滑出屏幕，透明度从 100% 渐变至 0%；
全局主导航栏：以 translateY(0) 为目标，从屏幕外向下平滑滑入，透明度从 0% 渐变至 100%。
第 150-300ms（同步收尾阶段）：
CTA 引导栏：完全滑出屏幕，透明度归 0，设置为 visibility: hidden；
主导航栏：完全滑入屏幕，透明度归 100%，恢复初始固定顶部状态，保持全页面可见。
四、补充交互规则（避免体验漏洞）
动画互斥：正向 / 反向动画执行期间，若用户反向滚动，立即终止当前动画，执行反向动画，避免动画冲突；
状态锁定：动画执行完成后，锁定当前导航栏状态，直至下一次滚动触发；
移动端适配：动画逻辑与桌面端完全一致，仅调整导航栏高度适配移动端屏幕；
性能优化：使用 will-change: transform, opacity 提前声明动画属性，保证动画 60fps 流畅运行，无卡顿。
五、前端可直接复用的专业术语（开发沟通用）

CTA 引导栏：Homepage CTA Bar / Sticky Action Banner

向下显示：slideInDown / translateY(0)

三、修改与添加功能设计（操作便捷，不破坏界面美观）
核心逻辑：“点击编辑→进入编辑状态→保存修改”，编辑入口隐蔽但易找到，编辑状态与查看状态区分明显，同时保证操作流畅，具体设计如下：
1. 编辑入口设计
- 模块级编辑：每个一级模块卡片的右上角，鼠标悬浮时显示“编辑”按钮（图标+文字，图标为铅笔，文字为“编辑”），点击后进入该模块的整体编辑状态，卡片边框变为主色调，底部显示“保存”“取消”按钮。
- 项级编辑：对于列表项、表格项、文本段落等具体内容，鼠标悬浮时，内容右侧显示小型“编辑”图标（仅图标，不显示文字），点击后仅激活该内容的编辑状态，不影响其他内容，避免整体编辑的繁琐。
- 添加入口：每个可添加内容的模块（如core_strengths、hard_skills_gaps），在模块底部显示“添加”按钮（图标+文字，图标为加号，文字为“添加”），按钮样式与编辑按钮一致，hover时高亮，点击后弹出添加表单。
1. 编辑状态设计（兼顾操作与美观）
- 文本类编辑（字符串、段落）：点击编辑后，文本区域变为输入框（单行/多行，根据内容长度自适应），输入框边框为细边框（主色调），背景为白色，padding 8px 12px，圆角6px；输入框大小与原文本区域一致，避免页面布局错乱；占位符提示“请输入内容”，字体为浅灰色，字号12px；输入框右侧新增“AI润色”小图标，点击可对当前输入的文本进行单独润色，润色后显示对比弹窗，可选择“应用润色”“放弃润色”。
- 列表类编辑（数组）：点击列表项编辑，弹出小型编辑表单（仅包含该列表项的输入框），表单采用卡片式，圆角8px，阴影轻微，与页面整体风格统一；点击“添加”按钮，弹出添加表单，表单包含该类数据的所有必填属性（如添加硬技能差距，需包含技能名称、重要性、当前水平等），表单字段按顺序排列，标签与输入框对应，间距8px，提交按钮与取消按钮并排，样式与顶部保存按钮一致；表单内每个文本输入框右侧均添加小型“AI润色”图标，支持单字段润色。
- 对象数组类编辑（如hard_skills_gaps）：每个小卡片的右上角添加“编辑”“删除”图标，点击编辑弹出表单（填充当前数据），点击删除弹出确认弹窗（弹窗采用圆角设计，包含“确认删除”“取消”按钮，避免误操作）；添加时，弹出完整表单，字段按属性顺序排列，重要性、当前水平等可选属性采用下拉框（下拉框样式统一，圆角6px，hover选项高亮）；表单文本输入区域右侧添加“AI润色”图标，支持针对性润色；小卡片hover时，除“编辑”“删除”图标外，新增“AI润色”按钮，点击可对整个卡片内容进行润色。
- 编辑反馈：修改内容后，未保存时，编辑入口变为“保存中”（灰色），保存成功后，显示短暂提示（绿色对勾+“保存成功”，位于模块右上角，3秒后自动消失），保存失败时，显示红色提示（叉号+“保存失败，请重试”）；取消编辑时，恢复原内容，无额外提示，避免干扰；AI润色反馈与编辑反馈风格统一，润色中显示加载提示，润色成功显示绿色对勾+“润色完成”，润色失败显示红色叉号+“润色失败，请重试”。
1. 内容校验设计
编辑、添加内容时，添加基础校验，避免无效数据，同时给出清晰提示：
- 必填项校验：如候选人姓名、目标岗位、技能名称等必填字段，未填写时，输入框边框变为红色，下方显示浅红色提示文字（如“请填写该内容”）。
- 格式校验：评分类字段（如skill_match_score）仅允许输入0-100的数字，输入非数字时，提示“请输入0-100的数字”；日期、时长类字段（如duration）按指定格式（如“1-6个月”）校验，格式错误时给出对应提示。
- 长度校验：文本类字段（如candidate_summary.current_background）设置最大长度（如500字），超出时提示“内容过长，请控制在500字以内”，同时输入框右侧显示当前字数/最大字数，方便用户控制。
四、PDF导出功能设计（便捷高效，导出样式美观）
核心要求：导出的PDF与前端展示样式一致，布局整齐、文字清晰，支持自定义导出选项，同时保证导出速度，具体设计如下：
1. 导出入口
顶部导航栏右侧，“保存”按钮旁边放置“导出PDF”按钮，按钮样式与保存按钮一致（主色调、圆角、hover放大），点击后弹出导出选项弹窗（避免误导出），弹窗简洁，包含导出范围、导出样式两个核心选项，底部为“确认导出”“取消”按钮。
1. 导出选项设计（满足个性化需求）
- 导出范围：提供两个选项（单选），“全部模块”（默认）、“当前模块”，用户可根据需求选择，避免导出无用内容；选项用radio按钮，样式统一，与弹窗整体风格适配。
- 导出样式：提供两个选项（单选），“保持前端样式”（默认）、“简洁打印样式”；保持前端样式：PDF布局、配色、字体与前端一致，保留卡片、阴影、高亮等设计；简洁打印样式：去除阴影、背景色，仅保留文字和基础边框，适配打印需求，节省墨水。
- 额外选项：可选“添加页码”“添加导出时间”（默认勾选），页码位于PDF底部居中，导出时间位于封面右上角，字体为小字号（10px），浅灰色，不影响整体美观。
1. 导出过程与反馈
- 导出加载：点击“确认导出”后，弹窗关闭，页面顶部显示加载进度条（主色调，高度3px，宽度自适应），同时显示提示文字“正在导出PDF，请稍候...”，进度条实时显示导出进度，提升用户体验。
- 导出完成：导出成功后，进度条消失，显示绿色提示（对勾+“PDF导出成功，已自动下载”），同时PDF文件自动下载（文件名默认“候选人职业报告-陈程.pdf”，可自定义文件名，在导出选项中添加“文件名输入框”）。
- 导出失败：若导出失败（如网络问题、内容过多），显示红色提示（叉号+“导出失败，请重试”），同时提供“重新导出”按钮，点击后重新触发导出流程，无需重新选择导出选项。
1. PDF样式优化（保证导出美观）
- 字体统一：PDF内字体采用宋体（正文）、黑体（标题），与前端字体一致，避免乱码，字号与前端保持比例（标题18px，正文14px，辅助文字12px）。
- 布局适配：PDF页面采用A4尺寸，纵向排版，内容自动分页，避免内容被截断；模块卡片在PDF中保留边框和圆角，阴影适当弱化，保证打印清晰；列表、表格内容在分页时，避免单个列表项、表格行跨页，提升可读性。
- 重点保留：前端中加粗、高亮的关键信息，在PDF中保持一致（如评分数字、核心建议）；颜色适配打印，主色调在PDF中转为深灰色，避免打印后颜色过浅，无法识别。
五、额外优化（提升美观度与用户体验）
- 响应式适配：PC端（屏幕宽度≥1200px）采用三段式布局；平板端（768px≤屏幕宽度<1200px）侧边栏可折叠，内容区卡片每行2个；移动端（屏幕宽度<768px）隐藏侧边栏，顶部导航栏添加“菜单”按钮，点击弹出侧边栏，内容区卡片每行1个，保证在小屏幕上的可读性和操作便捷性；AI润色弹窗在移动端自适应宽度（占屏幕宽度80%），按钮适配小屏幕尺寸，避免操作不便。
- 动画效果：添加轻微的过渡动画，如侧边栏折叠/展开（过渡时间0.3s）、编辑状态切换（淡入淡出）、按钮hover效果、提示信息弹出/消失（淡入淡出），提升交互质感，不夸张、不卡顿；新增AI润色相关动画，润色加载时显示旋转动画（与主色调一致），润色完成后显示淡入淡出的对比弹窗，润色按钮hover时轻微变色，与整体动画风格统一。
- 空状态处理：若某模块无数据（如未添加技能差距），显示空状态提示（图标+文字，图标为灰色，文字为“暂无相关内容，点击添加”），文字下方添加“添加”按钮，引导用户操作，避免页面空白，提升美观度；AI润色记录为空时，显示“暂无润色记录，点击AI润色生成”提示，下方添加“去润色”按钮，引导用户使用功能。
- 缓存功能：用户修改、添加内容后，自动缓存（本地存储），避免刷新页面后丢失数据；若用户关闭页面再重新打开，自动加载缓存的最新内容，同时显示提示“已加载上次编辑内容”，提升用户体验；新增AI润色记录缓存，自动保存最近10条润色记录（包含润色前后内容、润色时间），支持手动删除记录，避免缓存过多占用空间。
- 打印适配：支持直接打印页面，打印时自动切换为“简洁打印样式”，去除不必要的元素（如编辑按钮、导航栏），保证打印内容整齐、清晰，与PDF导出的简洁样式一致；AI润色相关按钮、弹窗不会被打印，避免干扰打印效果。
- AI润色细节优化：润色对比弹窗采用左右分栏布局，左侧“润色前”，右侧“润色后”，中间添加“切换视图”按钮，支持左右切换、上下对比；润色后的内容用浅底色标注，突出修改部分；支持“一键应用全部润色”“选择性应用润色”（点击修改部分单独应用）；润色设置弹窗可选择润色强度（轻度：仅优化语句流畅度；中度：优化语句+调整逻辑；重度：全面优化+补充细节），适配不同用户需求；润色功能支持多语言适配（默认中文，可切换英文），满足国际化需求。
六、注意事项（保证功能与美观兼顾）
- 编辑状态下，避免页面布局错乱，输入框、表单的大小和位置需与原内容对齐，不遮挡其他内容；AI润色弹窗、对比弹窗需居中显示，避免遮挡当前操作的内容，可拖动调整位置；规划类模块的折叠面板展开/收起时，避免页面整体跳动，添加0.3s过渡动画。
- 配色需统一，避免过多颜色堆砌，主色调、辅助色、文字色保持一致，提升整体协调性；AI润色相关元素（按钮、提示、动画）需贴合原有配色，辅助色仅用于突出AI润色入口，不新增其他颜色；规划类模块的状态标签颜色需与整体配色适配，不突兀。
- 导出PDF时，需保证所有内容都能正常显示，避免特殊字符、图标乱码，尤其是列表项、表格中的内容；AI润色后的内容导出时需与手动编辑内容保持一致，润色标注（浅底色）不导出，保证PDF整洁；规划类模块的进度条、状态标签导出时正常显示，颜色适配打印（状态标签颜色转为深灰系，保证打印清晰）。
- 操作按钮的样式需统一（圆角、尺寸、颜色），避免不同模块的编辑、添加、保存按钮样式不一致，影响美观；AI润色按钮（全局、模块级、字段级）样式统一，仅通过颜色区分优先级，确保视觉统一；规划类模块的“标记状态”“批量更新状态”按钮样式与整体按钮统一，大小适配内容。
- 大段文字、多个列表项的模块，需设置滚动条（仅模块内滚动），避免页面整体滚动过长，提升查看体验；AI润色对比弹窗若内容过长，需设置内部滚动条，方便查看完整的润色前后内容；规划类模块折叠面板展开后内容过多时，添加模块内滚动条，不影响页面整体布局。
- AI润色功能需添加权限控制，仅有权限的用户可使用（如管理员、编辑者），普通查看者隐藏AI润色入口；润色过程中需避免重复触发，添加防抖处理（点击润色按钮后1秒内不可重复点击），防止卡顿；规划类模块润色时，需严格保留核心规划数据，仅优化表述逻辑和流畅度。
- 润色内容需符合职业报告的专业调性，AI润色不可修改核心数据（如评分、学历、技能名称），仅优化语句表达、逻辑流畅度，避免润色后偏离原内容；润色完成后需保留原内容，支持用户对比后选择是否应用；规划类模块的润色需贴合规划的专业性，避免口语化表述，提升规划方案的严谨性；里程碑润色需重点优化表述的规范性和可执行性，不修改里程碑名称、完成时限、状态等核心数据。



职业报告前端编辑功能实现细节
编辑功能核心实现逻辑：遵循“隐蔽入口+分层编辑+实时反馈+数据校验+缓存保障”原则，适配不同类型数据（文本、列表、对象数组、嵌套层级、里程碑），兼顾操作便捷性与界面美观，与整体简约商务风格保持一致，具体细节如下：
一、编辑入口实现细节
1. 模块级编辑入口
- 入口位置：每个一级模块（如候选人总结、匹配分析、规划类模块等）卡片的右上角，默认隐藏，仅当鼠标悬浮在该模块卡片上时显示。
- 入口样式：采用“图标+文字”组合，图标为铅笔形状，文字为“编辑”，按钮与顶部导航栏按钮样式统一，圆角8px， hover时轻微放大（transform: scale(1.05)），颜色为主色调（#2E86AB）。
- 触发效果：点击后，该模块卡片边框切换为主色调细边框，卡片底部弹出“保存”“取消”两个按钮，按钮样式与入口按钮一致，间距12px，进入模块整体编辑状态，模块内所有可编辑内容同步激活。
2. 项级编辑入口
- 入口位置：针对列表项、表格项、文本段落、对象卡片等具体内容，入口位于内容右侧，默认隐藏，鼠标悬浮在对应内容上时显示。
- 入口样式：仅显示铅笔图标（无文字），图标颜色为中灰色（#666666），hover时变为主体色，尺寸与内容适配，不遮挡周边内容。
- 触发效果：点击后仅激活当前单个内容的编辑状态，不影响模块内其他内容，避免整体编辑的繁琐，编辑状态与内容区域对齐，不破坏页面布局。
3. 添加入口
- 入口位置：每个可添加内容的模块（如核心优势、硬技能差距、里程碑等）底部，居中显示，始终可见（空状态时与空提示文字配合显示）。
- 入口样式：“图标+文字”组合，图标为加号，文字为“添加”，按钮样式与编辑按钮统一，主体色，圆角8px， hover时高亮。
- 触发效果：点击后弹出对应内容的添加表单，表单为卡片式，圆角8px，轻微阴影，居中显示，可拖动，表单字段与对应内容的属性一一对应。
二、不同类型内容编辑实现细节
1. 文本类内容编辑（字符串、段落，如候选人背景、教育适配说明）
- 编辑组件：点击编辑后，文本区域替换为单行/多行输入框，输入框尺寸与原文本区域完全一致，避免页面布局错乱。
- 组件样式：输入框边框为主色调细边框（宽度1px），背景为白色，内边距8px 12px，圆角6px，字体与正文一致（14px，深灰色#333333）。
- 辅助提示：输入框内显示浅灰色（#999999）占位符“请输入内容”，字号12px；文本过长时，输入框自动换行（多行输入框），并显示滚动条（仅输入框内滚动）。
- 关联功能：输入框右侧新增小型“AI润色”图标（与编辑入口图标适配），点击可触发单字段润色，不影响编辑操作。
2. 列表类内容编辑（数组，如核心优势、待改进领域）
- 编辑形式：点击列表项右侧的编辑图标，弹出小型编辑表单，表单仅包含该列表项的输入框，卡片式设计，与整体风格统一。
- 添加表单：点击“添加”按钮弹出的表单，仅包含该列表项的必填字段（单个输入框），表单底部设置“提交”“取消”按钮，样式统一，间距8px。
- 交互细节：编辑时，输入框实时显示原有内容，修改后点击“提交”即时更新列表显示；取消编辑则恢复原内容，无额外提示；表单内输入框右侧添加“AI润色”图标，支持单字段润色。
3. 对象数组类内容编辑（如硬技能差距、职业路径阶段）
- 编辑入口：每个对象对应的小卡片右上角，除“编辑”“删除”图标外，hover时新增“AI润色”按钮，三个图标间距8px，样式统一。
- 编辑表单：点击“编辑”图标弹出表单，表单内填充该对象的所有属性（如技能名称、重要性、当前水平等），字段按逻辑顺序排列，标签为浅灰色（#999999）加粗，与输入框间距4px。
- 组件适配：可选属性（如重要性、当前水平）采用下拉框，下拉框样式统一，圆角6px，hover选项高亮，选项颜色与标签颜色一致；文本属性采用输入框，与文本类编辑输入框样式一致。
- 删除交互：点击“删除”图标，弹出确认弹窗，弹窗为圆角卡片，包含“确认删除”“取消”按钮，避免误操作，弹窗居中显示，可拖动。
4. 规划类模块嵌套层级编辑（如短期目标、行动步骤）
- 折叠项编辑：二级折叠项（如短期目标）标题右侧，hover时显示“编辑”“AI润色”按钮，点击“编辑”可修改折叠项标题及关联的核心信息（如时长、目标描述）。
- 三级数据编辑：三级数据项（如行动步骤、量化目标）hover时显示“编辑”“标记状态”“AI润色”三个图标，点击“编辑”弹出对应表单，表单字段与数据项属性对应。
- 状态联动：点击“标记状态”可快速切换状态（未开始→进行中→已完成），状态切换后，实时同步主卡片的进度概览进度条，无需额外保存。
- 批量操作：主卡片底部添加“批量更新状态”按钮，点击弹出状态选择弹窗，可统一修改多个三级数据项的状态，提升操作效率。
5. 里程碑专属编辑（重点优化，适配多场景）
（1）项级快速编辑
- 入口样式：里程碑数据项hover时，除原有三个图标外，新增“快速编辑”图标（铅笔+时钟组合），与其他图标间距8px，颜色为主色调。
- 编辑组件：点击后弹出小型快捷编辑框，仅包含“里程碑名称”“完成时限”两个核心字段，输入框样式与文本类编辑一致，编辑框圆角8px，背景白色，主色调边框，不遮挡周边内容。
- 提交反馈：编辑完成点击“确认”，实时更新里程碑显示内容，同步触发本地缓存，状态未变更则不影响进度概览；点击“取消”则放弃修改，恢复原内容。
（2）详情弹窗编辑
- 入口触发：点击里程碑项的“编辑”图标（非快速编辑），弹出详情编辑弹窗，弹窗宽度450px，居中显示，可拖动，卡片式设计，轻微阴影，内边距24px。
- 表单字段：包含里程碑所有核心字段（名称、完成时限、关联目标、具体要求、关联行动步骤、状态），字段按“基础信息→关联内容→状态”的逻辑排列。
- 特殊适配：“关联行动步骤”采用下拉选择，选项为当前规划模块内的所有行动步骤；“状态”采用下拉切换，选项为未开始/进行中/已完成，修改状态后实时同步进度概览。
- 底部按钮：弹窗底部设置“保存”“取消”“AI润色”三个按钮，间距12px，样式统一，点击“AI润色”可针对性润色里程碑表述，不修改核心数据。
（3）批量修改
- 入口位置：主卡片底部“批量更新状态”按钮旁，新增“批量修改里程碑”按钮，样式与批量更新状态按钮一致，主体色。
- 弹窗设计：点击后弹出批量修改弹窗，弹窗内显示所有里程碑列表，每个里程碑左侧添加勾选框，支持“全选”“取消全选”，勾选框样式统一。
- 修改字段：可统一修改“状态”“完成时限”两个共性字段，字段组件与详情编辑弹窗一致，修改后实时预览修改效果。
- 二次确认：点击“确认批量修改”后，弹出二次确认提示（“确认批量修改选中的X个里程碑？”），避免误操作，确认后所有勾选里程碑同步更新，进度概览重新计算。
三、编辑状态反馈实现细节
- 未保存反馈：修改内容后未点击保存时，编辑入口按钮变为灰色，显示“保存中”文字，提示用户需保存修改。
- 保存成功反馈：点击“保存”后，按钮恢复正常样式，模块右上角显示绿色对勾+“保存成功”提示文字，提示文字为浅灰色，3秒后自动淡入淡出消失，不干扰操作。
- 保存失败反馈：保存失败时，模块右上角显示红色叉号+“保存失败，请重试”提示文字，文字为浅红色，同时“保存”按钮变为红色，点击可重新尝试保存。
- 取消编辑反馈：点击“取消”按钮，编辑状态立即解除，所有未保存的修改自动恢复为原内容，无额外提示，避免冗余。
- 未保存退出反馈：修改内容未保存时，切换页面、关闭弹窗或刷新页面，弹出提示弹窗（“修改未保存，是否放弃？”），包含“放弃”“继续编辑”两个按钮，防止数据丢失。
四、内容校验实现细节
1. 必填项校验
- 校验范围：候选人姓名、目标岗位、技能名称、里程碑名称等必填字段，未填写时触发校验。
- 反馈样式：输入框边框变为浅红色，输入框下方显示浅红色提示文字（如“请填写该内容”），提示文字字号12px，与输入框间距4px。
- 提交限制：必填项未填写时，“保存”“提交”按钮处于禁用状态，禁用颜色为浅灰色（#DCDCDC），鼠标悬浮时显示“请完善必填项”提示。
2. 格式校验
- 评分类字段：如技能匹配分、经验匹配分等，仅允许输入0-100的数字，输入非数字、超出范围时，输入框边框变红，提示“请输入0-100的数字”。
- 时长/日期类字段：如里程碑完成时限、规划阶段时长，按“1-6个月”“2026-06”等指定格式校验，格式错误时提示“请按指定格式输入（如1-6个月）”。
- 校验时机：实时校验，用户输入过程中同步判断，输入正确后，错误提示自动消失，输入框边框恢复正常。
3. 长度校验
- 校验范围：文本类字段（如候选人背景、里程碑具体要求），设置最大长度（默认500字），可根据字段需求调整。
- 反馈样式：输入内容超出最大长度时，输入框边框变红，输入框右侧显示当前字数/最大字数（如“120/500”），提示文字“内容过长，请控制在500字以内”。
- 输入限制：超出最大长度后，禁止继续输入，同时光标定位在输入框末尾，提示用户删减内容。
五、编辑缓存实现细节
- 缓存触发：用户每完成一次编辑（点击“保存”），自动触发本地存储缓存，缓存内容为当前模块的最新编辑内容。
- 缓存加载：用户关闭页面后重新打开，系统自动加载本地缓存的最新编辑内容，同时在页面顶部显示“已加载上次编辑内容”提示，3秒后自动消失。
- 缓存清理：缓存自动保留最近10次编辑记录，超出后自动覆盖最早记录；用户可手动清除缓存（通过侧边栏设置入口，未显示在主界面，避免干扰）。
- 里程碑缓存：里程碑修改后，缓存同步更新，包含修改后的所有字段及状态，进度概览进度同步缓存，重新打开页面后进度不丢失。
六、编辑功能适配细节
- 响应式适配：PC端编辑表单、弹窗正常显示；平板端编辑表单自适应宽度（占屏幕宽度80%）；移动端编辑入口图标放大，表单适配小屏幕，避免操作不便。
- 动画适配：编辑状态切换、表单弹出/收起、提示信息显示/消失，添加0.3s淡入淡出过渡动画，避免页面跳动，提升交互质感。
- 打印/导出适配：编辑后的内容，导出PDF、打印时正常显示，编辑状态下的输入框、按钮不被导出/打印，保证导出/打印内容整洁。
